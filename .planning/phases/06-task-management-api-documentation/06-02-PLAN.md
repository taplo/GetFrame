    **IMPORTANT utoipa compatibility notes (from RESEARCH.md — follow exactly):**
    - Use `status = CODE` (not `status = "200"`) in response tuples
    - For empty response bodies (204), use `body = ()` or omit `body`
    - Ensure all response types used in `body = TypeName` derive `ToSchema`
    - Use `content_type = "application/json"` when overriding default
    - Always add a trailing comma after the last response tuple
    - The `#[utoipa::path(...)]` macro is picky — test compile after each file

    ### Step 1: Annotate request/response types with ToSchema

    **`src/config.rs`** — Add `#[derive(ToSchema)]` to:
    - `StreamConfig` (also needs `ToSchema import`)

    **`src/pipeline/rule.rs`** — Add `#[derive(ToSchema)]` to:
    - `RuleConfig` enum
    - `CompositeOperator` enum

    **`src/health.rs`** — Add `#[derive(ToSchema)]` to:
    - `HealthResponse`
    - `ReadyResponse`

    **`src/api/streams.rs`** — Add `#[derive(ToSchema)]` to:

    ### Step 2: Add #[utoipa::path] to handler functions

    **`src/health.rs`** — Health endpoints:
    - `health_handler`: GET /health, returns HealthResponse (200)
    - `ready_handler`: GET /ready, returns ReadyResponse (200) or 503

    **`src/api/streams.rs`** — Stream endpoints (document using task tag "streams"):
    - `list_streams`: GET /api/v1/streams, returns StreamListResponse (200)
    - `create_stream`: POST /api/v1/streams, body CreateStreamRequest, returns StreamResponse (201)
    - `get_stream`: GET /api/v1/streams/{id}, returns StreamResponse (200), 404
    - `update_stream`: PUT /api/v1/streams/{id}, body UpdateStreamRequest, returns StreamResponse (200), 404
    - `delete_stream`: DELETE /api/v1/streams/{id}, returns 204, 404
    - `test_connection`: POST /api/v1/streams/{id}/test, returns JSON body (200), 404

    **`src/api/rules.rs`** — Rule endpoints (tag "rules"):
    - `list_rules`: GET /api/v1/streams/{id}/rules, returns RulesResponse (200), 404
    - `create_rule`: POST /api/v1/streams/{id}/rules, body CreateRuleRequest, returns RuleOperationResponse (201), 404, 409
    - `get_rule`: GET /api/v1/streams/{id}/rules/{index}, returns RuleOperationResponse (200), 404
    - `update_rule`: PUT /api/v1/streams/{id}/rules/{index}, body UpdateRuleRequest, returns RuleOperationResponse (200), 404
    - `delete_rule`: DELETE /api/v1/streams/{id}/rules/{index}, returns 204, 404

    **`src/api/tasks.rs`** — Task endpoints (tag "tasks"):
    - `list_tasks`: GET /api/v1/tasks, returns TaskListResponse (200)
    - `create_task`: POST /api/v1/tasks, body CreateTaskRequest, returns TaskInfo (201)
    - `get_task`: GET /api/v1/tasks/{id}, returns TaskInfo (200), 404
    - `delete_task`: DELETE /api/v1/tasks/{id}, returns 204, 404
    - `start_task`: POST /api/v1/tasks/{id}/start, returns TaskInfo (200), 404, 409
    - `pause_task`: POST /api/v1/tasks/{id}/pause, returns TaskInfo (200), 404, 409
    - `resume_task`: POST /api/v1/tasks/{id}/resume, returns TaskInfo (200), 404, 409
    - `stop_task`: POST /api/v1/tasks/{id}/stop, returns TaskInfo (200), 404, 409

    ### Step 3: Create ApiDoc in src/api/mod.rs

    In `src/api/mod.rs`, add:
    ```rust
    use utoipa::OpenApi;

    #[derive(OpenApi)]
    #[openapi(
        info(
            title = "GetFrame API",
            description = "High-performance video frame extraction platform API",
            version = "0.1.0",
            license(name = "MIT")
        ),
        paths(
            // Health
            crate::health::health_handler,
            crate::health::ready_handler,
            // Streams
            crate::api::streams::list_streams,
            crate::api::streams::create_stream,
            crate::api::streams::get_stream,
            crate::api::streams::update_stream,
            crate::api::streams::delete_stream,
            crate::api::streams::test_connection,
            // Rules
            crate::api::rules::list_rules,
            crate::api::rules::create_rule,
            crate::api::rules::get_rule,
            crate::api::rules::update_rule,
            crate::api::rules::delete_rule,
            // Tasks
            crate::api::tasks::list_tasks,
            crate::api::tasks::create_task,
            crate::api::tasks::get_task,
            crate::api::tasks::delete_task,
            crate::api::tasks::start_task,
            crate::api::tasks::pause_task,
            crate::api::tasks::resume_task,
            crate::api::tasks::stop_task,
        ),
        components(schemas(
            // Health
            crate::health::HealthResponse,
            crate::health::ReadyResponse,
            // Streams
            crate::api::streams::StreamResponse,
            crate::api::streams::StreamStatusResponse,
            crate::api::streams::CreateStreamRequest,
            crate::api::streams::UpdateStreamRequest,
            crate::api::streams::StreamListResponse,
            // Rules
            crate::api::rules::RulesResponse,
            crate::api::rules::CreateRuleRequest,
            crate::api::rules::UpdateRuleRequest,
            crate::api::rules::RuleOperationResponse,
            // Tasks
            crate::api::tasks::TaskListResponse,
            crate::api::task_types::TaskInfo,
            crate::api::task_types::TaskStatus,
            crate::api::task_types::CreateTaskRequest,
            // Config
            crate::config::StreamConfig,
            // Rules
            crate::pipeline::rule::RuleConfig,
            crate::pipeline::rule::CompositeOperator,
        ))
    )]
    pub struct ApiDoc;
    ```

    **BUT** — `TaskInfo`, `TaskStatus`, `CreateTaskRequest` are defined in `src/task/mod.rs` (or registry), not in `src/api/tasks.rs`. Since utoipa needs the types to be in scope and accessible, either:
    - Re-export them from `src/api/tasks.rs` (add `pub use crate::task::{TaskInfo, TaskStatus, CreateTaskRequest};`) so they can be referenced as `crate::api::tasks::TaskInfo`, or
    - Reference them directly as `crate::task::TaskInfo`.
    
    Use direct re-exports in `src/api/tasks.rs`:
    ```rust
    pub use crate::task::{TaskInfo, TaskStatus, CreateTaskRequest, TaskId};
    ```

    Also need `#[derive(ToSchema)]` on the task types in `src/task/mod.rs` or `src/task/registry.rs`. Add `use utoipa::ToSchema;` and `#[derive(ToSchema)]` to `TaskInfo`, `TaskStatus`, and `CreateTaskRequest`.
